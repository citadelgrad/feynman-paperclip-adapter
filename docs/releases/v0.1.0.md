# v0.1.0

Initial public release of the Feynman Paperclip adapter.

## Highlights
- Adds a `feynman_local` adapter for running Feynman as a managed Paperclip agent.
- Resolves the local Feynman installation and launches Pi with Feynman’s prompts, tools, and runtime context.
- Streams structured run output back into Paperclip and preserves session continuity across runs.
- Fixes Paperclip execution reliability by running Pi in JSON mode instead of RPC mode.
- Fixes repo-context issues by inferring the real project cwd from `instructionsFilePath` when Paperclip only has an `agent_home` fallback workspace.
- Improves Feynman discovery in restricted service environments by augmenting PATH with common user bin directories such as `~/.local/bin`.

## Notes
- `@samfp/pi-memory` works on macOS after rebuilding `better-sqlite3` with Feynman’s bundled Node/npm.
- `@kaiserlich-dev/pi-session-search` is currently disabled in the tested Rosco setup because the published package appears incomplete upstream.
